# Бэкенд для хранилища изображений с иерархическими тегами и поиском

## Описание
Пользователи с помощью REST API могут загружать изображения, присваивать им иерархические теги 
и проводить поиск по этим тегам. Поддерживаются базовые CRUD-операции, автодополнение по 
префиксу тега, перемещение тега под другой родительский тег.

## Технологический стек
- **Язык:** Golang
- **Фреймворк:** Fiber
- **База данных для хранения пользователей и тегов:** PostgreSQL (библиотека pgx)
- **Иерархия**: ltree (расширение для PostgreSQL)
- **Хранилище для изображений:** Garage S3
- **Документация API:** Swagger (библиотека swag)
- **Миграции:** goose 
- **Контейнеры:** Docker, docker-compose


## Требования
Для запуска:
- **Docker** и **docker-compose**
- **make**
- **bash**

Для разработки (дополнительно к тому что выше):
- **Go 1.26**
- **goose** (можно установить с помощью ```make install-goose```)
- **swag** (можно установить с помощью ```make install-swag```)

Первоначальная настройка проекта:
- сгенерировать .env файл: ```make setup-env```
- _(не обязательно)_ поменять нужные переменные среды

## Запуск и остановка
Запуск в фоне:
```bash
make run
```

Запуск с логами:
```bash
make run-and-attach
```

Остановка и удаление контейнеров:
```bash
make stop
```

## API
После запуска документация доступна на `localhost:8080/swagger`.
